## feature/user 주요 기능 
    - 사용자의 세션을 기반으로 이름으로 로그인 할 수 있다 
    - 도서를 장바구니에 담을 수 있다 
        - 장바구니는 로그인 한 사용자만 가능한 기능이다 .
        - 로그인 인증은 인터셉터가 가로채어 로그인 유무를 확인하고 진행된다
    
## 특이사항
    - 장바구니와 사용자 정보는 모두 메모리 저장소에 저장한다 
        - 서버가 꺼지면 모든 데이터가 사라진다 
    - 간단한 테스트를 위해 만들어진 장바구니와 사용자 정보 

## 주의 
    - 1. 로그인 시 별도의 회원가입 로직없이 이름으로 로그인 및 저장이 됨 
        - a. 로그아웃시 세션 초기화를 진행하여 기준이되는 세션 ID도 초기화되어 무한 가입이 가능함
    - 2. Cart 엔티티 와 연관관계인 CartItem 
        - a. Cart 1 - CartItem N 관계
        - b. CartItem N - Book 1 
        - c. User 엔티티가 따로 존재하지 않아 Cart에서 User와 연관되어 있는 필드가 String으로 
             단순 저장이 되어있음 , 객체 매핑이 아님 (연관 관계가 아님) 

    
## Junit 테스트 요약
    1. 비로그인 장바구니 담기 테스트 
    [POST] localhost:8080/cart/add?bookId=1&quantity=1
    로직 흐름 : 
            1. 해당 엔드포인트로 파라미터에 도서 ID와 수량을 전달한다 . 
            2. /cart 엔드포인트에 접근하기 전 인터셉터가 가로채어 로그인 유무를 확인함
                a. 로그인은 세션에 저장되어있는 username의 키값을 확인하여 존재한다면 로그인으로 간주한다
            3. username이 비어있어 요청한 엔드포인트 대신 /login 으로 강제 이동
    응답 결과 : redirect:/login
            - 로그인 페이지 렌더링 된다

                a. 로그인은 세션에 저장되어있는 username의 키값을 확인하여 존재한다면 로그인으로 간주한다
